Flask Blog Template Setup Guide

1. Fork the Repository

First, create your own copy of the repository:

  1. Visit https://github.com/bobbyconnolly/flask-learner-template
  2. Click the "Fork" button in the top-right corner
  3. Select your GitHub account as the destination

Forking gives you your own copy of the project to work with. This means you can make changes without affecting the original template.

2. Clone Your Fork

Clone your forked repository to your local machine:

git clone https://github.com/YOUR-USERNAME/flask-learner-template.git
cd flask-learner-template

Replace YOUR-USERNAME with your actual GitHub username

3. Set Up Virtual Environment

PyCharm should create and activate a virtual environment automatically when you open the project.

Make sure you're using a Python interpreter in PyCharm before proceeding.

4. Install Dependencies

Install required packages using pip:

pip install -r requirements.txt

This will install:

  • Flask==2.0.3
  • Werkzeug==2.0.3
  • flask-sqlalchemy==2.5.1

5. Initialize the Database

Run database.py to create the SQLite database and add sample data:

python database.py

This will:

  • Create the database file blog_database.db
  • Set up the required tables
  • Add sample blog posts

6. Start the Application

Run the Flask application:

python app.py

Visit http://127.0.0.1:5000/ in your browser

Available Routes

  • / - Home page, shows all posts
  • /post/<id> - View a specific post
  • /create - Create a new post
  • /edit/<id> - Edit an existing post
  • /delete/<id> - Delete a post

Features

  • Create, view, edit, and delete blog posts
  • Upload images and videos (max 16MB)
  • Supported media formats:
    • Images: PNG, JPG, JPEG, GIF
    • Videos: MP4, WebM, MOV
  • Responsive design that works on mobile

Project Structure

Key files:

  • app.py - Main Flask application
  • database.py - Database operations
  • templates/ - HTML templates
  • static/css/style.css - Styling
  • CODE_BLOCKS/ - Example code snippets

Need Help?

If you run into issues:

  • Check the CODE_BLOCKS folder for examples
  • Make sure all dependencies are installed
  • Verify the database was created successfully
  • Ask your instructor or classmates
